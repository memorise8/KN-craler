# 01. 프로젝트 개요

## 목적

Kidsnote(유치원/어린이집 사진 공유 서비스) 학부모 계정으로 로그인하면 자녀의 모든 앨범과 사진을 브라우저에서 조회하고, 로컬 디스크로 일괄 다운로드할 수 있는 **독립 실행형 풀스택 웹앱**.

기존 `frwaler/crawler/sites/custom/kidsnote.py` 커스텀 크롤러는 CLI 기반 + 다른 크롤러 프레임워크에 의존했기 때문에, **FastAPI + Vanilla JS SPA**로 떼어내어 누구나 브라우저만으로 사용할 수 있도록 재구성.

## 핵심 가치 제안

1. **별도 가입 불필요** — Kidsnote 계정으로 바로 로그인.
2. **서버에 자격증명 미저장** — 서버는 `sessionid` 쿠키만 메모리에 2시간 보관.
3. **백그라운드 다운로드** — 수백~수천 장도 비동기 스레드로 처리, 진행률 실시간 확인.
4. **재개 가능** — 파일명이 이미 존재하면 스킵하여 중단 지점 이후만 받음.
5. **빌드 불필요** — 프론트는 순수 HTML/JS/CSS, Node.js 툴체인 없이 바로 실행.

## 사용 시나리오

- **A. 특정 앨범만 빠르게** — 앨범 URL로 진입 → 갤러리 확인 → 다운로드 버튼.
- **B. 자녀 전체 백업** — 로그인 후 자녀 카드 클릭 → 상단 "전체 다운로드" 버튼.
- **C. 로그인 직후 자동 시작** — 로그인 화면의 "로그인 후 전체 자동 다운로드" 체크박스.

## 기술 스택

| 레이어 | 선택 | 이유 |
|---|---|---|
| 백엔드 | Python 3.11 + FastAPI + `requests` | 기존 크롤러와 언어 일치, 작은 풋프린트 |
| 작업 실행 | 표준 `threading` | I/O 바운드 + 단일 프로세스로 충분 |
| 세션 저장 | SQLite + 복원형 `KidsnoteClient` | 재시작 후 로그인 유지, 단일 프로세스 운영 기준 |
| 프론트 | Vanilla JS + CSS Grid | 빌드 불필요, 외부 의존성 0 |
| 배포 | `uvicorn` 단일 프로세스 | 로컬/소규모 배포 전용 |
